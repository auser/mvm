# Field-name tokens that mark a JSON Schema property as secret-shaped.
# Plan-0009 / ADR-0009. Conservative: matches whole-name or trailing
# `_token` / `-token` suffix forms, so `auth_strategy_name` does NOT
# trip but `api_key`, `auth_token`, `private_key`, etc. do.
#
# Adding a token causes function args/return schemas with matching
# field names to fail validation with E_SECRET_IN_SCHEMA.
# Lines starting with `#` and blank lines are ignored.
#
# Two tiers:
#   1. Auth credentials — the canonical secret-mount case.
#   2. Financial / government identifiers that are nearly always a
#      "should-have-been-a-secret" mistake when they appear as a
#      function-arg field. Broader PII (email, phone, address) is
#      intentionally NOT in this list — those have legitimate
#      function-arg uses and belong in a separate, lower-severity
#      check if we ever want one.
